# Versionamiento

ApoloPanel usa Semantic Versioning:

- `MAJOR`: cambios incompatibles.
- `MINOR`: nuevas funcionalidades compatibles.
- `PATCH`: correcciones compatibles.

## Política inicial

Mientras el proyecto esté en `0.x`, pueden existir cambios internos incompatibles. Aun así, cada release debe documentarse en `CHANGELOG.md`.

## Convención de releases

- `0.1.x`: base técnica y MVP visual.
- `0.2.x`: agente VPS funcional.
- `0.3.x`: dominios, sitios web y SSL.
- `0.4.x`: bases de datos y backups.
- `1.0.0`: versión estable para uso operativo.
